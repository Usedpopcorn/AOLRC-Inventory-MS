# UI Consistency Skill Source

Use this folder as the repo-side source package for consistent UI work on AOLRC Inventory.

## Start Here

- `UI_COMPONENTS.md`: what shared primitives exist and when to use them.
- `RESPONSIVE_UI_RULES.md`: breakpoint expectations and structural rules.
- `UI_REGRESSION_CHECKLIST.md`: required validation sweep.
- `AGENTS.md`: concise repo guidance for Codex and other agents.

## Shared Source Of Truth

- `templates/_ui_macros.html`
- `templates/_inventory_macros.html`
- `templates/_item_macros.html`
- `static/css/styles.css`

## Intended Use

Use this skill source when a task changes:

- page headers
- toolbars
- chips or state banners
- forms and settings flows
- inventory rows, family groups, singleton assets, or detail panels
- responsive layout behavior
- empty/no-results/loading/error states

Read `checklists.md` before editing and `examples.md` when deciding whether a local fix is acceptable.
