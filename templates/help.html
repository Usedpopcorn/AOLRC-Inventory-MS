Viewer

Admin

{% call ui.form_section("Quick Status vs Raw Counts", "The two tracking methods answer different operational questions.") %}

Quick Venue Status

Status is the day-to-day signal. It answers: How does this venue look right now? Staff use it when a visual check is enough to keep the team current.

{{ ui.status_pill("good", "Good", "bi-check-circle-fill") }} {{ ui.status_pill("ok", "OK", "bi-check-circle-fill") }} {{ ui.status_pill("low", "Low", "bi-exclamation-triangle-fill") }} {{ ui.status_pill("out", "Out", "bi-x-circle-fill") }} {{ ui.status_pill("not_checked", "Not Checked", "bi-dash-circle") }}
  • Best for current-day visibility and fast room sweeps.
  • Powers venue status badges on the Dashboard and Venue Profile.
  • Stored as status history, so activity can show who refreshed which item.

Raw Count Data

Counts are the audit and ordering record. They answer: How many do we actually have? Staff use raw counts when the number matters for par, restocking, trend review, or orders.

  • Best for long-term audit, real count tracking, and replenishment planning.
  • Feeds Supplies, Restocking, and Orders through current count and count history records.
  • A recorded zero means counted and confirmed empty; missing count means no current count was submitted.

Use status when speed matters.

Example: a staff member checks a venue and marks an item as Low so the Dashboard immediately shows attention is needed.

Use counts when accuracy matters.

Example: a staff member counts 12 units of a tracked item against a par of 20 so Supplies and Orders can calculate the restocking gap.

Singleton assets use presence language.

Singleton assets are marked Present, Damaged, or Missing instead of counted like consumable supplies.

Status mode

Tap only the status buttons for rows that changed. Unchanged rows keep their previous saved status. Save when the pending-change banner shows the updates you expect.

Raw Counts mode

Enter counts only for rows you actually counted. The page keeps the previous count visible as context, but a blank input does not overwrite existing data.

Notes

Staff can add item-specific notes from the row note button. Notes are useful when a status or count needs a reason, such as damaged equipment or a temporary supply move.

Missing count

No current count has been submitted for that venue/item pair. This is an audit gap, not a zero.

Recorded zero

Someone counted the item and confirmed none were on hand. Treat this as valid data and a likely restocking signal.

Coverage

Coverage shows whether counts exist across the venues where an item is actively tracked.

{% call ui.form_section("Troubleshooting and FAQ", "Start here when data looks surprising or controls are missing.") %}

Why did a status change but the count did not?

Status and raw counts are separate. A quick status update keeps the day-to-day signal current, but it does not overwrite numeric count data for quantity items.

Why does Supplies show Not Counted instead of 0?

Not Counted means no current raw count exists. A displayed 0 means someone entered zero and confirmed the item was empty.

Why can I see a page but not edit it?

Your role controls write access. Viewers are read-only, staff can update operational data and notes, and admins manage setup and orders.

Why is an item missing from a venue check?

Admins should verify the item is active, not only a family organizer, and assigned to the venue through Venue Settings or Tracking Setup.

What should I do before creating an order?

Review raw counts for the relevant venue or setup group, resolve obvious missing counts, then create the order batch so the frozen snapshots are meaningful.
